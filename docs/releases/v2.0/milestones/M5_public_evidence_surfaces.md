# Milestone 5 — Public Evidence Surfaces

**Tag:** `v2.0-M5`  
**Date:** 2026-03-17  
**Status:** COMPLETE  
**Preceding milestone:** M4 — Replay & Evidence (with corrections)

---

## Objective

Surface the evidence bundles generated by the Level 3 Replay Engine as first-class, publicly navigable pages on the SOaC Enterprise Portal. Every MTIP package now links directly to a rich, interactive evidence view — making replay verdicts, MITRE coverage tables, step timelines, and raw artifacts accessible to anyone visiting the site.

---

## Deliverables

### 1. Dynamic Evidence Route (`/packages/[id]/evidence`)

A new Next.js dynamic route renders the evidence bundle for any package:

```
/packages/pkg-001/evidence
/packages/pkg-002/evidence
…
/packages/pkg-011/evidence
```

**Tabs:**

| Tab | Content |
|---|---|
| Overview | Stats grid (steps, detections, playbook actions, coverage %), pillar breakdown bar chart, artifacts referenced |
| MITRE Coverage | Table of declared techniques with validation status, tactic, and exercising step numbers — linked to attack.mitre.org |
| Timeline | Full step-by-step replay timeline with pillar filter and expandable rows |
| Full Report | Rendered `replay-report.md` with simple markdown→HTML conversion |

Each page also provides direct links to the raw `evidence-manifest.json` and `replay-report.md` files.

### 2. Updated Package Cards

- **Evidence badge:** Each package card on `/packages` now shows a green `✅ VERIFIED` badge next to the package ID when its evidence verdict is PASS.
- **Evidence button:** The green "Evidence" button on each card now routes to `/packages/{id}/evidence` (the interactive viewer) instead of linking to the raw markdown file.

### 3. Registry Update (`data/packages.json`)

Each package object now includes an `evidence` block:

```json
"evidence": {
  "manifest": "/packages/001/evidence/evidence-manifest.json",
  "report": "/packages/001/evidence/replay-report.md",
  "verdict": "PASS",
  "coverage_pct": 100,
  "route": "/packages/pkg-001/evidence"
}
```

All 11 packages have `verdict: "PASS"` and `coverage_pct: 100` (scoped to declared techniques).

## Architecture Note

The evidence viewer is a purely client-side component that fetches the static JSON and Markdown files from `/packages/NNN/evidence/` at runtime. No API routes or database queries are involved — the evidence data is served as static assets from the `public/` directory. This means the evidence is always consistent with what the CI pipeline generates and what ships in the package ZIPs.
